### 语法/指示：
- for (为了)

- that (那个)
- on (在...之上)

- algorithm + compilation + rom：作为一个 Android 开发者，这一组是你的职业本能词。
- viscosity (粘度) vs elastic (弹性)：想象 UI 动画中的阻尼感和回弹效果。
- retrieve (检索)：这在 SQLite 或 API 调用中非常常见，与上一组的 retrieval 形成词性对应。
- sluggish (卡顿)：在 Android 性能优化中，主线程被耗时操作占据会导致 UI 变得 sluggish。
- choppy (掉帧)：描述 A/V SDK 在网络抖动或丢帧严重时，输出的画面和声音非常 choppy。
- clog (拥塞)：描述消息队列（Message Queue）或网络带宽被海量数据 clog 住了。
- waive (豁免)：在集成第三方库时，阅读 License 协议，看哪些责任被 waived (免除) 了。
- jerk (抖动)：在传感器数据处理或视频稳定（Stabilization）算法中，消除不自然的 jerk (突发性抖动)。
- Steep Memory Growth (内存激增)：在进行 LeakCanary 内存分析时，如果看到折线图有一个 steep 的上升趋势，那通常意味着发生了严重的内存泄漏。
- Steep Learning Curve for Rust (Rust 的学习曲线)：在 2026 年，如果你主导将 C++ 的 A/V 核心层重构成 Rust，需要向团队说明虽然初期学习曲线 steep，但长期的安全性收益巨大。
- Code Rot (代码腐化)：描述由于长期缺乏维护，代码库逐渐变得难以理解和运行的现象。
- Hurdle (技术门槛)：13 年经验的你，现在面临的 hurdle 可能是如何将 AI 模型高效集成到移动端 A/V SDK 中。
- Grunt work (苦力活)：即使是资深架构师，偶尔也要处理一些诸如手动适配旧机型或整理旧文档的 grunt work。
- Poke (探测/交互)：在调试硬件接口或 Socket 通信时，你可能需要 poke the interface (尝试与接口交互) 来观察反馈。
- Ethos (工程文化)：在德国工作，你会感受到严谨（Rigour）和标准化（Standardisation）的工程 ethos。


### 开发障碍：
- hurdle (障碍/跳栏)。在处理跨平台 A/V 协议兼容性时，最大的 hurdle 是不同平台的编解码差异。负面状态： rot (腐烂/堕落 - 如 Code Rot 代码腐化)
- sear (灼伤/使枯萎)
- lack (缺乏/不足)。









- 急剧下降： steep drop (急剧下降)。例如：A steep drop in frame rate during 4K 120FPS rendering. (4K 120 帧渲染时帧率急剧下降)。
- 衰减曲线： steep attenuation (陡峭衰减)。在音频滤波器（Low-pass filter）设计中，描述截止频率后的信号衰减斜率。

- 学习曲线： steep learning curve (陡峭的学习曲线)。13 年的资历让你深知，理解 AOSP 底层音视频驱动或最新的 WebGPU 渲染管线，其学习曲线是非常 steep 的。
- 高昂代价： steep price (过高的代价)。形容某些虽然功能强大但内存占用极高的第三方 SDK：The performance gain comes at a steep price in memory.

- steep rent (高昂的房租)。在讨论海外搬迁（Relocation）方案时，评估核心地段的学习或居住成本。
- steep street (陡峭的街道)。描述你未来在海外城市徒步或骑行时的地形体验。
