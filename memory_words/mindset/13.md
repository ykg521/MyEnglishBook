- socket (套接字)：底层网络编程（TCP/UDP）的核心组件。
-  credentials (凭据)：登录认证或 GitHub SSH 配置时必用的词汇。
- ruminate (沉思)：这个词很有趣
- 原意是牛“反刍”
- 在职场中常指对一个方案反复思考。
- silicon (硅)
- mechanics (机制/力学)：常用于描述系统或游戏的运行逻辑（Game Mechanics）。
- schematic (原理图)：硬件开发中必看的电路原理图。
- anomaly (异常)：在数据监控（Monitor）或日志分析中
- 检测 Anomaly 是核心任务。
- redundant (冗余)：分布式系统中
- Redundant nodes 是保证高可用的基础。
- Deterioration of Signal (信号恶化)：在音视频弱网优化中
- 描述音频抖动（Jitter）或视频质量 deterioration 的根源及应对策略。
- Episodic Latency (间歇性延迟)：在性能调优中
- 排查那些并非持续发生而是 episodic (间歇性) 的 GC 停顿或线程锁竞争。
- Marshal Threading (线程调度)：作为 13 年老兵
- 你的核心价值在于能够高效地 marshal 并行计算资源
- 确保 SDK 在低功耗下依然运行流畅。
- Materialise the UI (实现 UI)：将复杂的动效设计从稿件 materialise (具现化) 到 Android 自定义 View 中。
- Cliché in Code (代码陈词滥调)：面试时可以谈谈你如何通过重构去掉那些 cliché 且低效的样板代码
- 引入更现代的声明式 UI 或协程逻辑。
- Resource Leash (资源束缚)：在 2026 年的高端 Android 旗舰机优化中
- 为 SDK 挂上一条 leash（如通过 cgroups 或 OOM Score）
- 确保其后台活动不会导致前台 App 掉帧。
- Refactoring Clichés (重构陈词滥调)：13 年经验的你
- 价值往往体现在如何将那些 cliché 的嵌套回调（Callback Hell）重构成简洁的 Kotlin Coroutines。
- Data-Fed Latency (数据馈送延迟)：分析 A/V 渲染管线时
- 重点检查数据被 fed 进 Buffer 的速率与渲染时钟是否同步。
- Architecture Warts (架构瑕疵)：在技术委员会（Technical Committee）会议上
- 坦诚指出当前系统的 warts
- 并提出渐进式的迁移方案。
- Memory Bump (内存突刺)：在音视频录制过程中
- 检测到由于 Buffer 未及时回收导致的 memory bump
- 是 A/V SDK 性能监控的核心指标。
- Network Hop (网络跃点)：在进行全球实时音视频传输（RTC）链路优化时
- 减少中转服务器的 hops 能显著降低端到端延迟（Latency）。
- Code Cliché (代码陈词滥调)：面试时可以谈谈你如何通过重构去掉那些 cliché 的遗留逻辑
- 引入更简洁的 Kotlin DSL 提高代码可读性。
- Trump the Competition (超越竞争对手)：你主导研发的低延迟音频算法
- 在 2026 年的基准测试中成功 trumped 了市面上主流的竞品 SDK。
- Architecture Phalanx (架构方阵)：描述一个由多个紧密协作的服务组成的微服务群
- 它们像一个 phalanx 一样保证了系统的极高可用性。
- Eureka Moment (发现时刻)：在 A/V SDK 优化中
- 当你通过 Trace 终于定位到那个导致 2026 年新款手机掉帧的 Root Cause 时
- 那就是你的 eureka 瞬间。
- Buffer Brimful (缓冲区溢满)：在高并发推流场景下
- 如果队列 brimful
- 资深工程师需要决定是采取丢帧策略还是动态调整码率（ABR）。
- Code Cliché (代码陈词滥调)：在重构过程中
- 去掉那些 cliché 的样板代码（Boilerplate）
- 利用 Kotlin 扩展或更高级的 DSL 来精简系统。
- Swoop on Data (数据抓取)：描述你的监控系统如何在高负载瞬间 swoop 并捕获内存堆栈（Heap Dump）以供离线分析。
- Technical Squabble (技术争论)：在架构评审（Design Review）中
- 合理的 squabble 能帮助团队避开盲目引入新技术栈的坑。
- Pre-emptive Scheduling (抢占式调度)：作为 13 年老兵
- 你在优化 SDK 线程优先级时
- 必须深刻理解 Linux 内核的 pre-emptive 机制。
- Audio Treble (音频高音)：在 A/V SDK 的均衡器模块中
- 针对 treble 频段的动态增益控制是提升音质通透度的核心。
- Network Stammer (网络抖动)：在描述直播拉流卡顿时
- 用 network stammer 替代简单的 "lag"
- 能更专业地描述非持续性的微小抖动。
- Forge a Protocol (定制协议)：为了绕过防火墙或提升传输效率
- 你曾主导 forged 了一套基于 UDP 的私有加密协议。
- Image Pyramid (图像金字塔)：在视频超分辨率或缩放算法中
- 利用 pyramid 结构进行多尺度特征提取是标准的 A/V 处理流程。
- Stencil Buffer (模板缓冲)：在 Android OpenGL/Vulkan 开发中
- 利用 stencil 缓冲实现复杂的遮罩裁剪特效
- 是高级 A/V SDK 渲染引擎的标配。
- Code Mileage (代码效用)：评估一个公共组件的 mileage
- 看它能在多少个业务线中稳定运行。
- Scrupulous Code Review (审慎的代码审查)：作为 13 年经验的专家
- 你不仅要关注逻辑
- 更要 scrupulously 检查代码中的安全性与可维护性。
- Architecture Behemoth (架构巨兽)：在 2026 年的重构计划中
- 你的任务是将那个运行了十年的 behemoth 单体系统拆分为微服务或 KMM 模块。
- Beacon Discovery (信标发现)：在物联网或多设备互联 SDK 中
- 处理蓝牙 beacon 的扫描与连接是实现“全家桶”生态的基础。
- Audio Propagation (音频传播)：在 A/V SDK 空间音频（Spatial Audio）开发中
- 模拟声波在不同材质间的 propagation 是核心算法之一。
- Deteriorate Performance (性能退化)：13 年经验的你
- 需要能够敏锐察觉由于 GC 频繁引发的 UI 渲染质量 deteriorate。
- Sketchy Logic (不可靠逻辑)：在 Code Review 中
- 指出那些 sketchy 的多线程同步逻辑
- 防止在生产环境出现 Race Condition（竞态条件）。
- Incinerate Data (数据销毁)：在符合 GDPR 规范的 SDK 开发中
- 确保用户隐私数据被彻底 incinerated 而非仅仅标记删除。
- Verbalise Technical Debt (表述技术债)：作为 Staff 工程师
- 你不仅要写代码
- 更要学会向管理层 verbalise 解决技术债的必要性与紧迫感。
