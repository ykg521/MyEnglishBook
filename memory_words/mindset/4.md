# words











### 词根与逻辑：
- pollinate (授粉) -> pollinator (授粉媒介)。
- frustrate (挫败) -> frustrated (感到挫败的)。
- industrial -> preindustrial (前工业时代的)。
- discrepancy (差异)
- sameness (相同性)
- otherness (异质性)
- cliché (陈词滥调)
- odds (几率/不和 - 如 Odds of success 成功率)
- whichever (无论哪个)
- either (两者之一)
- maybe (可能)
- then (然后)。
- commonsense (常识)
- thrift (节俭)
- serene (宁静的)
- plausible (似乎合理的)
- triumphantly (成功地)
- plainspoken (直言不讳的)
- predestine (预定/注定 - 类似于代码中的 Predefined)
- needless (不必要的 - 常用于优化：Needless re-rendering 无谓的重绘)
- till (直到/耕作)。
- waiver (豁免/放弃)
- readiness (就绪状态)
- recoup (重获/补偿)
- cancel (取消)
- scope (关联词)
- girder (大梁 - 常指架构支撑)。
- smaller (更小的)
- contrived (人为的/矫揉造作的 - 常指过度设计的架构)
- presentational (表现形式的 - 如 MVP/MVVM 中的 View 层)
- e.g. (例如)
- fourfold (四倍的)
- scarcity (匮乏)
- whether (是否)
- whatsoever (无论如何)
- e.g. (例如)
- alibi (不在场证明)
- humid (潮湿的)：悉尼或东京夏天的气候特点。
- drench (湿透)：被雨淋透。
- wispy (纤细的/模糊的) / dotted (有点的/星罗棋布的)。
- pinnacle (尖塔/顶点) / valley (山谷) / sill (窗台)。
- jupiter (木星)：天文词汇。
- Vanilla (原生)：作为 Android 老兵，你一定知道 Vanilla 版本的系统意味着没有厂商的 OEM 修改。
- Nuance (微妙)：在处理 A/V 同步（Lip-sync）时，毫秒级的差异就是 nuance。
- Nonetheless (转折)：The algorithm is complex; nonetheless
- it is highly efficient.
- vet (代码审查)：作为资深架构师，你的核心工作之一就是 vet the technical proposals (审查技术提案)。
- rear (后置)：在 A/V SDK 中，处理 Rear Camera 的流数据采集和镜像翻转。
- ruinous (性能崩坏)：不合理的线程池配置可能会对 App 的响应速度造成 ruinous effects。
- palatable (代码审美)：重构的目标是让代码变得更 palatable (优雅、易读)。
- pains (匠心)：描述你如何 take pains 优化 SDK 的启动耗时。
- skew (偏移)：在音视频领域，Audio-Video Skew 是你必须通过算法解决的同步难题。
- fetch (拉取)：这是每个程序员最熟悉的动作，无论是从网络 API fetch data 还是从本地缓存检索。
- rusty (生疏)：当你在 2026 年准备德国面试时，如果很久没写 Java，记得提前练手以免 get rusty。
- stuffed (填充)：描述数据结构中被无效数据填充（Padding）的状态。
- craft (工艺)：高级开发不仅是写代码（Coding），更是在雕琢作品（Crafting software）。


- refutation (反驳/辩驳)：在技术方案评审或 Code Review 中，用事实来反驳某种错误的假设。
- criticise (批评/评论)：对代码质量或架构设计的客观评价。
- prop (支撑物/道具)：在开发中常指 UI 的 Properties (属性) 简写，或系统的支撑组件。
- what (什么) / no. (编号/数字)：基础逻辑标识。





- emblazon (装饰/赞美) / emblem (徽章/象征)：品牌 Logo 或系统图标。
- pictures (图片) / fur (皮毛) / feather (羽毛)：渲染引擎中的材质描述。
- grey (灰色的)：UI 设计中的置灰态（Disabled state）。
- frightful (可怕的) / haunted (闹鬼的/忧心忡忡的)：形容那些逻辑诡异、难以调试的“鬼魂代码”。
- cannibal (食人者/同类相食)：在商业中指 Cannibalization (产品自相残杀)，比如新 SDK 抢占了旧产品的市场。


- vet (审查/诊治)：既指兽医，也指动词“严格审查”。在 Senior 职位面试中，公司会 vet 你的过往项目架构设计。
- no. (编号/数字)：number 的缩写。
- pains (辛劳/刻苦)：take great pains to do something (费尽心力做某事)。


- palatable (美味的/可接受的)：在技术选型中，指方案是否“易于被团队接受”。
- ruinous (毁灭性的/破坏性的)：形容严重的内存泄漏或安全漏洞对系统的影响。
- dying (垂死的/渴望的)：形容进程即将被系统回收（Process death），或极度渴望某种新技术。
- went (去/变得 - go 的过去式)：描述系统状态的演变。




- ufo (不明飞行物)：未知事物。
- pow (砰/啪)：象声词，表示撞击或突然的动作。
