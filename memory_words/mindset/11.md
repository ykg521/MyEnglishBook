- schema (架构)：在 Android 开发中
- 无论是数据库 SQLiteOpenHelper 还是 XML Layout
- 你都在处理各种 Schema。
- normalise (标准化)：在处理数据库数据或音视频 SDK 采样率时
- 经常需要做 Normalization。
- volatile (易挥发的/易变的)：在 Java/Kotlin 中
- volatile 关键字是保证并发可见性的核心
- 这里指其原意“易变的”。
- procrastinate (拖延)：这是程序员在面对复杂逻辑重构时最常见的行为（哈哈）。
- Detrimental (损害)：在 13 年的 A/V SDK 经验中
- 你一定深知 Main Thread Blocking (主线程阻塞) 对 Frame Rate (帧率) 是 detrimental 的。
- Antiquated (陈旧)：在 2026 年
- 迁移那些 antiquated 的 Java 层 A/V 逻辑到 Rust 或最新的 C++ 标准是资深开发者的价值所在。
- Readiness (就绪)：在多媒体 Pipeline 中
- 确保每一个 Component (组件) 的 readiness 是开始推流的前提。
- Recoup (回收/补偿)：在内存管理中
- 如何 recoup 泄露的 Native 内存是保持 SDK 长期运行稳定的关键。
- Waiver (豁免)：在参与开源项目或签署商业合同（如针对海外客户）时
- 理解 liability waiver (免责条款) 是职业素养的体现。
- Exception Catching (异常捕获)：在 A/V SDK 中
- 确保在 JNI 层充分 catching 所有的错误
- 防止底层崩溃导致 Java 层直接 Crash。
- Slough Off Legacy (去旧迎新)：13 年经验的资深架构师
- 价值往往体现在如何安全地 slough off 那些影响性能的陈旧逻辑。
- Cache Busting (缓存刷新)：在处理网络封面图或 CDN 流地址时
- 使用 cache busting 技术确保用户看到的是最新内容。
- Hungry for Innovation (技术渴望)：在 2026 年的面试中
- 展现你作为老兵依然 hungry for the latest AI-driven A/V technologies。
- Hone (精炼)：13 年经验的你
- 在 2026 年面试资深架构师时
- 重点应放在你如何 hone your expertise 在音视频同步算法或低延迟传输协议上。
- Race Conditions (竞态/混乱)：在多线程开发中
- 不当的锁机制会导致内存状态变成一团 jumble。
- Neat Architecture (整洁架构)：资深工程师的标志是写出 neat and maintainable code
- 而非炫技式的复杂代码。
- Needlessly Complex (不必要的复杂性)：在 Code Review 中
- 你的价值在于指出那些 needless complexity 并通过重构降低维护成本。
- Precedable (优先级)：在处理 A/V 渲染队列或线程池调度时
- 定义任务的 precedable 顺序是性能优化的核心。
- Memory Heap (堆内存)：在进行 13 年资深级别的 Android 性能优化时
- 分析 Heap Dump 寻找内存泄漏是你的拿手好戏。
- Tidal Traffic (潮汐流量)：描述后端或 SDK 在高峰时段（如晚间直播高峰）面临的周期性流量 tidal waves。
- Riddle the Bug (解谜)：面试时可以描述你如何通过复杂的日志追踪
- 最终解开了那个困扰团队数周的底层 riddle。
- Cope with Legacy (应对遗留系统)：在 2026 年的面试中
- 强调你如何高效地 cope with legacy C++ code
- 并逐步迁移到更现代化的框架。
- GNP (国民生产总值)：在讨论海外迁移（如德国或澳洲）的宏观经济背景或行业薪资水平时
- 可能会涉及该背景。
- Sizzle (高频噪声)：在 A/V SDK 的音频算法优化中
- 消除由于编码压缩或采样率转换导致的 sizzle noise 是提升音质的关键。
- Gear (传动/工具)：描述你的开发工具链
- 例如使用 custom-built CI/CD gear 来加速 SDK 的自动化测试。
- Reflexion (反射)：在计算计算机视觉（Computer Vision）或渲染管线中
- 处理光线的 reflexion 以增强视频特效的真实感。
- Hinge (关键点)：在技术架构设计中
- 指出某一个模块是整个系统稳定性（Stability）的 hinge (关键点)。
- Ultraclean (超净)：描述你对代码质量的追求——ultraclean code
- 不仅无 Bug
- 且具备极高的可读性与可维护性。
