# words








### 流体与渗透：
- permeation (渗透)
- percolate (过滤/渗透/滴漏 - 描述数据逐渐同步的过程)
- overfill (装得太满/溢出)

### 工具与设备：
- snorkel (潜水呼吸管)
-  cage (笼子)
-  Parcel (包裹/数据包 - Android 核心进程间通信机制 Parcelable 的词根)

### 风险、稳定与代价
- blunder (大错/失误)
- wrack (破坏/失事)
- stakes (赌注/利害关系)
- hinder (阻碍)
- precariously (不安全地/不稳定地)
- doom (毁灭/注定)
- shaking (关联词)
- fury (狂怒/剧烈)
- demolition (拆除/破坏)

### 抽象：
- aura (气氛/光环)
- angelic (天使般的)
- debut (初次登台/问世)
- lyric (歌词/抒情诗 - 音视频 SDK 开发中的核心 metadata)。处理歌词同步（LRC 解析）、精准对齐（Lyric Alignment）是播放器开发者的基本功。
- hyphen (连字符)：在 Android 开发的资源命名（如 kebab-case）中非常常见。
- nullify vs finalise：想象代码逻辑，一个使变量失效，一个将流程定稿。
- dopamine (多巴胺)：当你写完一个完美的逻辑块（Code block）时大脑分泌的物质。
- swastika (卐字)：由于你考虑移民德国，请务必注意这个词在德国法律中是禁止出现的符号（除非用于历史研究），是非常严肃的社会禁忌。
- Notch (刘海屏)：作为 Android 开发者，你必须处理 Display Cutout (刘海屏适配)，确保 UI 不被 notch 遮挡。
- Deprecated (弃用)：13 年经验的你一定见过无数被 deprecated 的 API（如旧的 Camera 接口），你需要引导团队迁移到 CameraX。
- Parcel (数据包裹)：在 Android 的 Binder 通信中，所有的对象都要序列化进一个 Parcel 进行传递。
- Refurbish (重构/翻新)：当 Legacy Code (遗留代码) 难以维护时，你的任务是 refurbish the architecture 以适应新的需求。
- Holistically (全局架构)：资深工程师不仅改一个 Bug，而是 holistically 思考这个 Bug 对整个 SDK 稳定性的影响。
- Brick (砖)：在 Android A/V 底层开发中，如果 Bootloader 或 Kernel 层面处理不当，设备就会变成 brick。
- Lite (轻量化)：13 年经验的你，一定主导过 App 的 Lite version 开发，旨在为低端机型减少资源占用。
- Accompanying (伴随现象)：在定位 A/V 同步 Bug 时，观察 CPU 飙升是否为 accompanying symptoms (伴随症状)。
- Abused (滥用)：Senior 工程师经常需要在 Code Review 中指出那些被 abused 的设计模式（如过度使用 Singleton）。
- Shank (连接部)：在处理某些特种音视频硬件（如内窥镜 SDK）时，可能需要描述探头 shank 的物理尺寸。
